# Constraints & Rules
1.  **Язык комментариев:** Все комментарии в коде должны быть на **русском языке**, подробно объясняя логику работы функций.
2.  **Структура кода:** Используй типизацию (type hints) и docstrings.
3.  **Обработка данных:** Обязательно используй `@st.cache_data` для функций загрузки CSV, чтобы избежать перезагрузки данных при взаимодействии с UI.
4.  **Безопасность:** Не используй жестко закодированные пути, используй относительные пути относительно корня проекта.
5.  **Визуализация:** Графики должны быть интерактивными (Plotly).
6.  **Ошибки:** Предусмотри обработку случаев, когда файлы данных отсутствуют или повреждены (try-except + st.error).
